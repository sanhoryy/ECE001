Windows:

Windows NT architecture, a set of operating systems produced and sold by Microsoft, is a multi-layered design consisting of two main components, user mode and kernel mode. It is a proactive multi-tasking operating system that is designed to work with single-processor and multi-processor (SMP) computers. To handle I / O requests, they use packet I / O, which uses asynchronous I / O request packets (IRPs) and asynchronous I / O. Beginning with Windows XP, Microsoft began making 64-bit versions of Windows available. Before that, there were only 32-bit versions of these operating systems.

User mode programs and subsystems are limited in terms of system resources that they can access, while kernel mode has unrestricted access to system memory and external devices. Kernel mode in Windows NT has full access to the computer's system and hardware resources. The Windows NT kernel is a hybrid kernel; Architecture includes a simple kernel, hardware abstraction layer (HAL), drivers, and a set of services (collectively called Executive), all in kernel mode.

User mode in Windows NT consists of subsystems capable of passing I / O requests to device drivers in the appropriate kernel mode using I / O Manager. The Windows NT user mode layer consists of "environment subsystems", which run applications written for many different types of operating systems, and "integrated subsystem", which performs system specific functions on behalf of environment subsystems. Kernel mode prevents user mode services and applications from accessing critical operating system areas that they should not have access to.

Executive interfaces deal with all user-mode subsystems with I / O, object management, security, and process management. The kernel is located between the hardware abstraction layer and the executive to provide multi-processor synchronization, thread, transmission, and interrupt schedule, trap processing, and exceptions. The kernel is also responsible for configuring device drivers upon boot. Kernel mode drivers exist in three levels: top-level drivers, medium drivers, and low-level drivers. The Windows Driver Model (WDM) is located in the middle layer and is primarily designed to be binary and compatibility between source between Windows 98 and Windows 2000. The lower level drivers are either old Windows NT device drivers that directly control the device or can Be plug and play (PnP) device bus.
